---
title: "check-property-names: validate @property tag names"
description: "Ensures @property tag names in JSDoc are not duplicated within the same block and that nested properties have properly defined root properties."
keywords: ["eslint", "jsdoc", "check-property-names", "typedef", "property"]
---

The `check-property-names` rule ensures that `@property` (or `@prop`) tag names within a JSDoc block are not duplicated, and that nested property paths (like `cfg.foo`) have a corresponding root property (`cfg`) defined first.

## Rule details

| | |
|---|---|
| Recommended | ✓ |
| Fixable | 🔧 Yes—auto-removes duplicate `@property` entries (when `enableFixer` is `true`) |
| Rule name | `"jsdoc/check-property-names"` |
| Context | Everywhere |
| Tags | `@property` (alias: `@prop`) |

## Options

<ParamField path="enableFixer" type="boolean" default="false">
  When `true`, automatically removes duplicate `@property` entries. Duplicates
  are identified by name only—the second occurrence is removed regardless of
  whether the type or description differs.
</ParamField>

## Examples

<Tabs>
  <Tab title="Incorrect">
```javascript
/**
 * @typedef {SomeType} SomeTypedef
 * @property Foo.Bar
 */
// Message: @property path declaration ("Foo.Bar") appears before any real property.

/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 * @property Foo.Bar
 */
// Message: @property path declaration ("Foo.Bar") root node name ("Foo")
//          does not match previous real property name ("foo").

/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 * @property foo
 */
// Message: Duplicate @property "foo"

/**
 * @typedef {SomeType} SomeTypedef
 * @property cfg
 * @property cfg.foo
 * @property cfg.foo
 */
// Message: Duplicate @property "cfg.foo"
```

With `enableFixer: true`:

```javascript
/**
 * Assign the project to a list of employees.
 * @typedef {SomeType} SomeTypedef
 * @property {string} employees[].name - The name of an employee.
 * @property {string} employees[].name - The employee's department.
 */
// Message: Duplicate @property "employees[].name"
// (The second entry is automatically removed)
```
  </Tab>
  <Tab title="Correct">
```javascript
/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 */

/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 * @property bar
 */

/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 * @property foo.foo
 * @property bar
 */

/**
 * @typedef {SomeType} SomeTypedef
 * @property foo
 * @property foo.bar
 * @property foo.baz
 * @property bar
 */

/**
 * Assign the project to a list of employees.
 * @typedef {SomeType} SomeTypedef
 * @property {object[]} employees - The employees responsible for the project.
 * @property {string} employees[].name - The name of an employee.
 * @property {string} employees[].department - The employee's department.
 */

/**
 * @typedef {SomeType} SomeTypedef
 * @property {Error} error Exit code
 * @property {number} [code = 1] Exit code
 */

/**
 * @class
 * @property {Error} error Exit code
 * @property {number} [code = 1] Exit code
 */
function quux(code = 1) {
  this.error = new Error('oops');
  this.code = code;
}
```
  </Tab>
</Tabs>

### Multiple typedefs in one comment

The rule handles multiple `@typedef` blocks within the same comment and scopes duplicate detection per typedef:

```javascript
/**
 * @typedef {object} foo
 * @property {string} type
 *
 * @typedef {object} bar
 * @property {number} type
 */
// Valid: "type" is defined once per typedef
```

## Configuration

```javascript
// eslint.config.js
import jsdoc from 'eslint-plugin-jsdoc';

export default [
  {
    plugins: { jsdoc },
    rules: {
      'jsdoc/check-property-names': ['warn', {
        enableFixer: true,
      }],
    },
  },
];
```

<Note>
This rule is included in `jsdoc.configs['flat/recommended']` and reports at the `warn` level by default.
</Note>
