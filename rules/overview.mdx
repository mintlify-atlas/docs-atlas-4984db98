`eslint-plugin-jsdoc` provides over 66 rules for validating, enforcing, and styling JSDoc comments in JavaScript and TypeScript projects.

Rules marked with ✓ are included in the `recommended` configuration. Rules marked with 🔧 support auto-fixing via `--fix`.

## Validation rules

These rules verify that the content and structure of JSDoc tags are correct.

| Rule | Description | ✓ | 🔧 |
|------|-------------|---|---|
| [check-access](/rules/check-access) | Validates `@access` tag values and prevents mixing access-control tags | ✓ | |
| [check-alignment](/rules/check-alignment) | Reports invalid alignment of JSDoc block asterisks | ✓ | 🔧 |
| [check-param-names](/rules/check-param-names) | Checks `@param` names match function parameter names | ✓ | 🔧 |
| [check-property-names](/rules/check-property-names) | Checks `@property` names are not duplicated and have defined roots | ✓ | 🔧 |
| [check-tag-names](/rules/check-tag-names) | Reports invalid JSDoc block tag names | ✓ | 🔧 |
| [check-types](/rules/check-types) | Reports invalid type casing and enforces preferred type names | ✓ | 🔧 |
| [check-values](/rules/check-values) | Validates values of `@version`, `@since`, `@license`, `@author`, and more | ✓ | |
| [valid-types](/rules/valid-types) | Requires all types and namepaths to be syntactically valid | ✓ | |
| check-examples | **(deprecated)** Validates JavaScript code within `@example` tags | | |
| [check-indentation](/rules/check-indentation) | Reports invalid padding inside JSDoc blocks | | |
| [check-line-alignment](/rules/check-line-alignment) | Reports invalid alignment of JSDoc block lines | | 🔧 |
| check-syntax | Reports syntax not valid for the configured mode | | |
| check-template-names | Checks that `@template` names are actually used in type aliases | | |
| [no-undefined-types](/rules/no-undefined-types) | Prohibits types not specified as globals or within `@typedef` | ✓ | |

## Configuration

To use the recommended configuration, add the following to your ESLint config:

```javascript
import jsdoc from 'eslint-plugin-jsdoc';

export default [
  jsdoc.configs['flat/recommended'],
];
```

For TypeScript projects:

```javascript
import jsdoc from 'eslint-plugin-jsdoc';

export default [
  jsdoc.configs['flat/recommended-typescript'],
];
```

To configure individual rules:

```javascript
import jsdoc from 'eslint-plugin-jsdoc';

export default [
  {
    plugins: { jsdoc },
    rules: {
      'jsdoc/check-access': 'warn',
      'jsdoc/check-alignment': 'error',
      'jsdoc/check-param-names': ['error', { enableFixer: true }],
    },
  },
];
```
