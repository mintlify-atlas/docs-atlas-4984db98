---
title: "Get started with eslint-plugin-jsdoc"
description: "Learn how to configure eslint-plugin-jsdoc with a recommended preset and start linting JSDoc comments in your JavaScript or TypeScript project in minutes."
---

eslint-plugin-jsdoc integrates with ESLint to enforce documentation quality across your codebase. This guide walks you from installation to your first lint run, using the recommended flat config setup for ESLint 9+.

<Steps>
  <Step title="Install the plugin">
    Install ESLint and eslint-plugin-jsdoc as dev dependencies.

    <CodeGroup>
    ```bash npm
    npm install --save-dev eslint eslint-plugin-jsdoc
    ```
    ```bash yarn
    yarn add --dev eslint eslint-plugin-jsdoc
    ```
    ```bash pnpm
    pnpm add --save-dev eslint eslint-plugin-jsdoc
    ```
    </CodeGroup>

    <Note>
      Requires Node.js `^20.19.0 || ^22.13.0 || >=24` and ESLint `^7.0.0 || ^8.0.0 || ^9.0.0 || ^10.0.0`.
    </Note>
  </Step>

  <Step title="Create your ESLint config">
    Create or update `eslint.config.js` with a recommended preset.

    <Tabs>
      <Tab title="JavaScript (flat config)">
        ```js eslint.config.js
        import {jsdoc} from 'eslint-plugin-jsdoc';

        export default [
          jsdoc({
            config: 'flat/recommended',
          }),
        ];
        ```
      </Tab>
      <Tab title="TypeScript (flat config)">
        ```js eslint.config.js
        import {jsdoc} from 'eslint-plugin-jsdoc';

        export default [
          jsdoc({
            config: 'flat/recommended-typescript',
          }),
        ];
        ```
      </Tab>
      <Tab title="Declarative (flat config)">
        ```js eslint.config.js
        import jsdoc from 'eslint-plugin-jsdoc';

        export default [
          jsdoc.configs['flat/recommended'],
        ];
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Write a JSDoc comment">
    Add a JSDoc comment to a function. The plugin checks that your documentation matches the actual code:

    ```js example.js
    /**
     * Adds two numbers together.
     * @param {number} a - The first number.
     * @returns {number} The sum.
     */
    function add(a, b) {
      return a + b;
    }
    ```

    The `recommended` config will flag this function because the `b` parameter is undocumented — `jsdoc/require-param` requires a `@param` tag for every parameter.
  </Step>

  <Step title="Run ESLint">
    Run ESLint against your source files:

    ```bash
    npx eslint .
    ```

    For the example above, you would see output like:

    ```
    example.js
      7:18  warning  Missing JSDoc @param "b" declaration  jsdoc/require-param

    ✖ 1 problem (0 errors, 1 warning)
    ```

    Fix the issue by adding the missing `@param` tag:

    ```js example.js
    /**
     * Adds two numbers together.
     * @param {number} a - The first number.
     * @param {number} b - The second number.
     * @returns {number} The sum.
     */
    function add(a, b) {
      return a + b;
    }
    ```
  </Step>
</Steps>

## Choosing the right preset

eslint-plugin-jsdoc ships several preset configs. Choose based on your project type:

| Project type | Config to use |
|---|---|
| JavaScript | `flat/recommended` |
| TypeScript (`.ts` files) | `flat/recommended-typescript` |
| JavaScript with TypeScript-style JSDoc | `flat/recommended-typescript-flavor` |
| TSDoc / TypeDoc | `flat/recommended-tsdoc` |
| Mixed JS + TS | `flat/recommended-mixed` |

Add `-error` to any config name (e.g., `flat/recommended-error`) to report all issues as errors instead of warnings.

<Tip>
  For TypeScript projects, `flat/recommended-typescript` automatically turns off rules that are redundant when TypeScript provides the types, such as `require-param-type` and `require-returns-type`.
</Tip>
